# Teachat

## 项目简介
`Teachat` 是一个面向零星小微作业的任务型团队社交应用探索。

它希望把“讨论”“方案”“执行”“技能评价”组织成一个协作流程，适合各种形式的团队在任务场景中共同决策与作业。

核心目标：
- 支持任务发起、方案提交、讨论表态、执行记录
- 结合家庭与事业两条身份线，构建长期贡献与成长路径
- 以茶话会故事化场景表达协作礼仪与社交规则

## 核心理念
- 任务型社交：任务既要“说”，也要“做”
- 双维身份：`&家庭` 与 `$事业` 并行，兼顾家族关系与团队事业
- 结构化表达：任务、方案、讨论、表态、风险、技能均以预设要素记录
- 公平评价：通过“品味”“随机双盲蒙评”等机制增强公正与礼貌
- 技能成长：记录作业过程、物资与技艺，形成个人与组织的能力档案
- 历史参考：需求方可从过往案例排序中获取参考数据，辅助做出更明智的商品/服务选择
- 成果可见：解题者的思考与执行过程完整记录，使小微事业贡献被更合理发现与评价

## 关键概念

### 结构演进设想
- 个人和家庭作为“关系层”的底层基础，不再作为行动主体
- 团队（创建或加入）成为真正的活动主体，承担发起、协作、讨论、任务、通知、消息等角色
- 因此系统可拆分为两层：
  1. “关系层”
     - 个人：身份、关系、偏好、私密信息
     - 家庭：亲属关系、家族上下文、情感/生活关系
     - 说明：个人和家庭不可关注，不可接发消息
  2. “行动层”
     - 团队：是真正的活动主体，承担发起、协作、讨论、任务、通知、消息等角色
     - 说明：团队之间可相互关注，可相互接发消息

### 身份与关系
- `&家庭`：代表家庭层面的关系链与成员视角
- `$事业`：代表事业/团队层面的协作节点与项目关系
- 每个用户可在家庭和事业两个维度中参与不同任务与讨论

### 活动类型

#### 说：讨论与决策
- 发帖时由“我觉得 / 出主意”引导明确立场
- 回复时需要选择“支持 / 反对”，不允许模糊立场
- 术语说明：
  - `茶围`：任务发起或目标设定
  - `茶台`：针对任务提出的方案或项目
  - `茶议`：围绕方案展开的讨论、记录与进展条目
  - `品味`：对某个茶议进行立场表态

#### 做：执行与作业
- 记录的要素包括：地点、物资、技能、魔法、风险、隐患
- `茶技`：解决问题的技艺方法，包含六个步骤：约茶、看看、脑火、建议、物资、动手
- `慎独`：强调流制设计应避免个人/团队进入无监护作业状态，保持责任意识与行为规范
- `公道`：强调评价机制应公平、合理、可解释

### 故事映射
本应用以“星际茶话会”设定为背景：
- 在星际飞船上，乘客在茶棚中围绕任务开展讨论与作业
- 茶棚规则：每个新议程只能一次“品味”，关键内容需经过随机双盲评审

## 术语表
- `茶团`：协作组织，可视为家庭或团队
- `集团`：多个茶团的集合，构成一个更大的协作组织，有最高管理团队和成员团队
- `茶围`：任务或目标发起
- `茶台`：方案或项目响应
- `茶议`：方案讨论与进度记录
- `品味`：对讨论或方案的表态
- `茶技`：解决问题的方法与操作记录
- `黄牌`：见证者对违规个人行为的警告
- `红牌`：见证者对严重违规个人行为的驱逐离场处罚

## 角色表
- `船长`：实例运营者（代理人）
- `见证者`：考场监考员，由覆盖高中低收入家庭的基础教育工作者随机抽选担任。所有见证者归属于系统预设的「见证者集团」。
- `茶博士`：茶棚工作（服务）人员，负责维护茶棚秩序与规则执行
- `茶客`：参与者，既是需求方也是解题者，既是评论者也是被评论者
- `同行`：在同一领域内具有相似背景或专业技能的个体
- `专家`：在某个领域内具有较高专业水平的个体

## 目录结构
- `main.go`：项目入口，负责配置加载、路由注册与服务启动
- `public/`：静态资源目录，包含 CSS、JS、图片、帮助文件
- `templates/`：Go 模板文件目录，用于页面渲染
- `DAO/`：数据访问层，定义数据库结构、CRUD 操作与数据包装
- `Route/`：路由层，负责 HTTP 处理、响应输出及共用辅助函数
- `sql/`：数据库脚本，包含建表语句与初始数据
- `util/`：通用工具函数包

## 运行指南
### 环境要求
- Go 1.18+
- PostgreSQL
- 浏览器访问 Web 页面

### 快速启动
1. 克隆项目
2. 创建数据库并执行脚本：
   - `psql -f sql/schema.sql`
   - `psql -f sql/seed_data.sql`
   - `psql -f sql/tea_paymeng_system.sql`
3. 配置 `config.json` 中的数据库连接与服务地址
4. 启动应用：
   - `go run main.go`
5. 在浏览器中访问 `config.json` 中配置的地址

### 运行提示
- 若希望局域网可访问，可将 `config.json` 中 `Address` 设置为 `0.0.0.0:8000`
- 若出现数据库连接错误，请检查 PostgreSQL 服务与 `config.json` 配置
- 运行时可查看控制台日志以定位模板、路由或数据库问题

## VS Code 开发建议
- 安装 `Go for VSCode` 插件及 `gopls`
- 打开项目后可使用 `Run -> Start Debugging` (F5)
- 使用 VS Code 终端执行 `go run main.go` 或 `go build` 进行测试

## 调试与网络说明
- macOS 防火墙可能影响外部访问，必要时允许 VS Code 或应用程序通过防火墙
- 确保服务绑定地址与访问 IP 匹配
- 检查防火墙状态：`sudo pfctl -s info | grep Status`

## 项目来源与改动
本项目源于《Go Web Programming》Sau Sheong Chang 的教学示例 `chitchat`。

主要改动：
1. 将原 `data` 目录改为本地 `DAO` 目录，包名改为 `dao`
2. 将路由处理从 `main` 包拆分到 `Route` 目录，形成独立 `route` 包
3. 将 `util` 单独提取为工具包
4. 升级 Bootstrap 3 和 jQuery 到 2024 年版本
5. 合并注册布局模板，统一页面布局
6. 将项目名改为 `teachat`，贴合茶话会故事环境
7. 根据业务设定扩展数据模型、路由与模板

## 设计补充说明
- `慎独`：强调个人与团队在无监督条件下保持行为规范与责任意识
- `公道`：指公平合理的评价体系，兼顾外行观点、同行看法与专家分析

> 当前数据和业务设置主要为尚未实现的“公道评价”功能做准备。该功能是一个复杂的评价工程，需要排查利益相关的家族/团队干扰，并实现个人与团队排序（排行榜）。
